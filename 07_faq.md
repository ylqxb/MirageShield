# 常见问题FAQ

## 部署类问题

### 1. 系统无法启动

**问题**：执行 `python start_server.py` 后，系统无法启动

**解决方案**：
- 检查端口是否被占用：`netstat -an | findstr :5000`（Windows）或 `lsof -i :5000`（Linux）
- 检查依赖是否安装：`pip list` 确认所有依赖已安装
- 检查配置文件是否正确：查看 `config/` 目录下的配置文件
- 查看日志：`tail -f logs/system.log` 查看错误信息

### 2. 依赖安装失败

**问题**：执行 `pip install -r requirements.txt` 时失败

**解决方案**：
- 确保网络连接正常
- 使用国内镜像源：`pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple`
- 升级 pip：`pip install --upgrade pip`
- 检查 Python 版本是否符合要求（3.8+）

### 3. 蜜罐部署失败

**问题**：尝试部署蜜罐时失败

**解决方案**：
- 检查 Docker 是否正常运行：`docker ps`
- 确保用户有权限访问 Docker：`sudo usermod -aG docker $USER`（Linux）
- 检查网络连接是否正常
- 查看系统日志：`tail -f logs/agent.log` 查看错误信息

### 4. 系统启动后无法访问

**问题**：系统启动成功，但浏览器无法访问 http://localhost:5000

**解决方案**：
- 检查防火墙设置，确保 5000 端口开放
- 检查系统是否绑定到正确的 IP：`netstat -an | findstr :5000`
- 尝试使用 `http://127.0.0.1:5000` 访问
- 查看系统日志：`tail -f logs/system.log` 查看错误信息

### 5. 环境变量配置错误

**问题**：系统提示环境变量配置错误

**解决方案**：
- 检查 `.env` 文件是否存在
- 确保 `.env` 文件格式正确
- 检查环境变量值是否符合要求（如密码长度、密钥格式等）
- 重启系统使环境变量生效

## 功能类问题

### 1. 智能体状态显示为 inactive

**问题**：智能体状态显示为 inactive

**解决方案**：
- 检查智能体配置文件是否正确
- 点击 "启动" 按钮重启智能体
- 检查系统资源是否充足
- 查看智能体日志：`tail -f logs/agent.log` 查看错误信息

### 2. 威胁检测无结果

**问题**：系统运行一段时间后，威胁检测无结果

**解决方案**：
- 检查网络连接是否正常
- 检查智能体状态是否为 active
- 调整防护级别，提高检测灵敏度
- 查看系统日志：`tail -f logs/system.log` 查看错误信息

### 3. 硬件管理页面无设备

**问题**：硬件管理页面显示无设备

**解决方案**：
- 检查系统是否有权限访问硬件设备
- 检查硬件设备是否正常连接
- 点击 "刷新" 按钮重新扫描设备
- 查看系统日志：`tail -f logs/system.log` 查看错误信息

### 4. 社区联防无法连接

> **说明**：社区联防功能为设计规划，当前版本可能未完全开放，具体以实际界面为准。

**问题**：社区联防显示连接失败

**解决方案**：
- 检查网络连接是否正常
- 检查社区服务器是否可访问
- 检查社区联防配置是否正确
- 查看系统日志：`tail -f logs/system.log` 查看错误信息

### 5. 系统性能下降

**问题**：系统运行一段时间后，响应变慢

**解决方案**：
- 检查系统资源使用情况：`top`（Linux）或任务管理器（Windows）
- 清理系统日志：进入 "系统设置" → "日志管理" → "清理日志"
- 优化系统配置：调整智能体扫描频率
- 考虑升级硬件

## 安全类问题

### 1. 密码忘记

**问题**：忘记管理员密码

**解决方案**：
- 修改环境变量 `ADMIN_PASSWORD` 设置新密码
- 重启系统后，使用新密码登录

### 2. 敏感数据泄露

**问题**：担心敏感数据泄露

**解决方案**：
- 确保系统使用 HTTPS 加密传输
- 检查 `.gitignore` 文件，确保敏感文件不会被提交
- 定期备份数据
- 查看系统日志，检查是否有异常访问

### 3. 系统被攻击

**问题**：系统检测到攻击

**解决方案**：
- 查看威胁详情，了解攻击类型
- 部署蜜罐进一步分析攻击者
- 将攻击 IP 加入黑名单
- 调整防护策略，提高防护级别
- 查看系统日志，分析攻击路径

### 4. 数据备份失败

**问题**：系统备份失败

**解决方案**：
- 检查备份目录权限：`chmod 755 backups/`
- 确保磁盘空间充足
- 检查网络连接是否正常（如果备份到远程）
- 查看系统日志：`tail -f logs/system.log` 查看错误信息

### 5. 系统被入侵

**问题**：怀疑系统被入侵

**解决方案**：
- 立即断开网络连接
- 备份系统日志和数据
- 检查系统文件完整性
- 重新部署系统
- 分析入侵原因，加强防护措施

## 技术类问题

### 1. 系统架构

**问题**：MirageShield 的系统架构

**解决方案**：
- 控制平面：策略引擎、安全评估、分层防御
- 数据平面：真实数据池、诱饵数据池、虚拟网络层
- 智能体系统：Prober（探路者）、Baiter（诱饵手）、Watcher（守望者）
- API 与 Web 界面：RESTful API、Web 管理界面

### 2. AI 智能体实现

**问题**：AI 智能体是如何实现的

**解决方案**：
- 内部智能体：核心功能通过内部代码实现
- 外部 API 集成：部分高级功能支持通过 API 调用外部服务
- 模块化设计：智能体之间通过消息队列进行通信
- 安全性考虑：采用加密传输保护数据安全

### 3. 数据存储

**问题**：系统数据如何存储

**解决方案**：
- 配置数据：存储在 `config/` 目录下的 JSON 文件
- 用户数据：存储在 `data/` 目录下的 JSON 文件，使用 AES-256 加密
- 日志数据：存储在 `logs/` 目录下的日志文件
- 备份数据：存储在 `backups/` 目录下

### 4. 网络通信

**问题**：系统如何保证网络通信安全

**解决方案**：
- 使用 SSL/TLS 加密传输
- 实现基于角色的访问控制
- 使用代理（如 Tor）确保匿名性
- 实现水印和蜜令牌技术

### 5. 性能优化

**问题**：如何优化系统性能

**解决方案**：
- 硬件优化：使用 SSD 存储，增加内存，使用多核 CPU
- 软件优化：启用缓存机制，优化数据库查询，使用异步编程
- 网络优化：优化网络拓扑，使用 CDN 加速静态资源
- 配置优化：调整系统参数，优化数据库配置
